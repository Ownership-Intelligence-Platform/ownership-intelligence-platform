在银行录入新客户信息时，name scanning（姓名扫描）是指对客户姓名进行自动化筛查和比对的流程。其主要目的是检测客户姓名是否与已知的高风险名单（如制裁名单、反洗钱黑名单、政治公众人物名单等）存在匹配或相似情况。

核心流程包括：

银行系统会将新客户的姓名与全球或本地的制裁、反洗钱、反恐名单进行比对。
使用模糊匹配算法（如拼写变体、同音异写等）提高识别准确率，避免漏检。
一旦发现潜在匹配，系统会自动提示合规或风控人员进行进一步人工审核。
这是客户尽调（CDD/KYC）流程中的关键环节，有助于防范金融犯罪和合规风险。
简而言之，name scanning 是银行合规系统自动检测客户姓名是否涉及风险名单的技术手段

这段逻辑相当于：

内部扫描：看看这个名字和现有 Entity 是否“像是同一个或相关客户”（防止重复开户、关联客户等）；
外部扫描：看看本地名单里有没有高风险名字（模拟 sanction / PEP / 黑名单）。

在真实银行场景中：

name scanning 通常指：把客户名称、证件信息等送去制裁名单 / PEP / 本地黑名单系统进行匹配；
会考虑多语言、多音字、别名、证件号、出生日期等；
命中后会产生一个 Screening Case，由合规人员人工审核。

5. 下一步可以怎么增强？
   如果你觉得这个方向 OK，后续可以逐步增强：

增加更多字段参与扫描：证件号、生日、国家地区等；
增加打分/阈值：例如 score >= 2 的才展示为“可能命中”，低于则只作为“弱相关”；
记录审计日志：每次 name scanning 的输入、结果、操作员选择结果都落库，便于审计；
与 CDD 报告联动：当开户时触发 name scanning 命中高风险名单，可以自动在 CDD 报告里加一节“名单扫描结果摘要”。
